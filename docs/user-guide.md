> 마지막 업데이트: 2025-01-30
> 버전: 1.0

## 1. 시작하기

### 1.1 시스템 소개

HR 의사결정 지원 시스템(HR-DSS)은 AI 기반으로 HR 관련 의사결정을 지원하는 시스템입니다. 자연어 질문을 입력하면 데이터 기반의 분석 결과와 3가지 대안을 제시하며, 모든 주장에는 근거가 연결됩니다.

**핵심 기능**
- 자연어 질문 입력
- 3가지 대안 자동 생성 (보수적/균형/적극적)
- As-Is vs To-Be 영향 시뮬레이션
- 근거 기반 검증 (환각 방지)
- HITL(Human-in-the-Loop) 승인 워크플로

### 1.2 지원 유스케이스

| 유스케이스 | 설명 | 예시 질문 |
|-----------|------|----------|
| **A-1: Capacity 분석** | 12주간 가동률 병목 예측 | "향후 12주간 AI팀의 가동률 병목은?" |
| **B-1: Go/No-go** | 신규 기회 수주 의사결정 | "이 프로젝트를 수주해도 될까요?" |
| **C-1: 증원 분석** | 인력 충원 타당성 분석 | "개발자 2명 증원이 타당한가요?" |
| **D-1: 역량 갭** | 역량 갭 분석 및 해소 방안 | "데이터 엔지니어링 역량 갭을 분석해줘" |

4. **검증**: 환각 위험도 및 검증 결과
5. **KG 뷰**: Knowledge Graph 시각화

## 3. 사용 방법

### 3.1 질문하기

**Step 1: 시나리오 선택 (선택사항)**

4가지 시나리오 프리셋 중 선택하면 기본 제약조건과 질문 템플릿이 자동 설정됩니다.

| 시나리오 | 기본 제약조건 |
|---------|-------------|
| Capacity 분석 | 기간: 12주, 가동률 임계치: 90% |
| Go/No-go | 성공률 임계치: 60%, 최소 ROI: 15% |
| 증원 분석 | 최대 증원: 5명, 가동률 임계치: 85% |
| 역량 갭 | 최소 레벨: 3, 우선순위: 전략적 역량 |

**Step 2: 제약조건 설정 (선택사항)**

필요시 추가 제약조건을 설정합니다.

```
조직: AI/ML팀
기간: 2025-01-01 ~ 2025-03-31
예산 한도: 5,000만원
우선순위: 비용 절감
```

**Step 3: 질문 입력**

자연어로 질문을 입력합니다. 구체적일수록 정확한 분석이 가능합니다.

| 좋은 질문 | 개선이 필요한 질문 |
|----------|------------------|
| "향후 12주간 AI팀의 가동률 병목 구간과 해결 방안은?" | "인력 문제 해결해줘" |
| "프로젝트 A를 수주해도 될까요? 성공 확률과 리스크는?" | "프로젝트 수주 가능?" |

**Step 4: 결과 확인**

AI가 분석 결과와 3가지 대안을 제시합니다.

### 3.2 대안 비교하기

**레이더 차트 해석**

```
         영향도 (높을수록 좋음)
            ↑
            │
  비용 ←────┼────→ 실현가능성
 (낮을수록 좋음)    (높을수록 좋음)
            │
            ↓
         리스크 (낮을수록 좋음)
```

- 바깥쪽으로 넓을수록 해당 지표가 좋음
- 단, 비용과 리스크는 안쪽이 좋음

**대안 선택 가이드**

| 상황 | 추천 대안 |
|------|----------|
| 리스크를 최소화하고 싶을 때 | 보수적 대안 |
| 균형 잡힌 접근이 필요할 때 | 균형 대안 |
| 최대 효과를 원할 때 | 적극적 대안 |
| 빠른 실행이 필요할 때 | 보수적 또는 균형 |
| 장기적 성장을 원할 때 | 적극적 대안 |

### 3.3 근거 확인하기

모든 주장에는 근거가 연결되어 있습니다. 근거 패널에서 확인할 수 있습니다.

**근거 유형**

| 아이콘 | 유형 | 설명 |
|--------|------|------|
| [DATA] | 데이터 포인트 | HR 시스템, TMS 등에서 조회한 실제 데이터 |
| [CALC] | 계산 결과 | 모델 기반 계산/예측 결과 |
| [HIST] | 과거 사례 | 유사 프로젝트/의사결정 이력 |
| [RULE] | 규칙/정책 | 회사 정책, 업계 기준 등 |

**신뢰도 해석**

| 신뢰도 | 해석 | 조치 |
|--------|------|------|
| 90% 이상 | 높은 신뢰도 | 그대로 활용 가능 |
| 70-90% | 보통 신뢰도 | 주요 가정 검토 권장 |
| 70% 미만 | 낮은 신뢰도 | 추가 데이터 확인 필요 |

### 3.4 승인 요청하기

대안을 선택한 후 승인을 요청합니다.

**Step 1: 대안 선택**

3가지 대안 중 하나를 선택합니다. AI 추천 대안(★)을 참고할 수 있습니다.

**Step 2: 승인 요청**

"승인 요청" 버튼을 클릭합니다.

**Step 3: 승인자 지정**

승인 레벨에 따라 적절한 승인자에게 요청이 전달됩니다.

| 의사결정 유형 | 기본 승인 레벨 | 상향 조건 |
|-------------|--------------|----------|
| Capacity | 팀장급 | 갭 5 FTE 이상 시 부서장급 |
| Go/No-go | 부서장급 | 10억 이상 시 본부장급 |
| 증원 | 부서장급 | 5명 이상 시 본부장급 |
| 역량 갭 | 팀장급 | 전략적 역량 시 부서장급 |

**Step 4: 승인 대기**

승인자의 검토를 기다립니다. 승인/거절/수정 요청이 올 수 있습니다.

## 5. FAQ

### Q1: 질문을 해도 "알 수 없는 질문 유형"이라고 나와요.

**A:** 질문에 핵심 키워드가 포함되어야 합니다.

| 유스케이스 | 필수 키워드 |
|-----------|------------|
| Capacity | 가동률, 병목, 12주, capacity |
| Go/No-go | go/no-go, 수주, 성공 확률 |
| 증원 | 증원, headcount, 채용 |
| 역량 갭 | 역량 갭, competency, 스킬 부족 |

### Q2: 근거 연결률이 낮게 나와요.

**A:** 다음을 확인하세요.
1. 데이터가 최신인지 확인 (Data Quality 대시보드)
2. 질문한 조직/기간의 데이터가 있는지 확인
3. 관리자에게 데이터 보강 요청

### Q3: 대안 추천이 우리 상황에 맞지 않아요.

**A:** 제약조건을 더 구체적으로 설정하세요.
- 예산 한도 명시
- 우선순위 설정 (비용 절감 vs 효과 최대화)
- 리스크 허용 수준 설정

### Q4: 승인 요청이 에스컬레이션됐어요.

**A:** 다음 중 하나에 해당할 수 있습니다.
1. 원래 승인자가 권한 부족
2. 고영향 의사결정으로 자동 상향
3. 승인자가 수동으로 에스컬레이션

### Q5: 시뮬레이션 결과가 현실과 다른 것 같아요.

**A:** 시뮬레이션은 휴리스틱 기반 예측입니다.
- "가정" 탭에서 사용된 가정 확인
- 과거 유사 사례와 비교
- 실제 결과를 피드백하여 모델 개선 요청

## 6. 용어 사전

| 용어 | 설명 |
|------|------|
| FTE | Full-Time Equivalent, 풀타임 환산 인력 수 |
| HITL | Human-in-the-Loop, 인간 검토/개입 |
| KG | Knowledge Graph, 지식 그래프 |
| 가동률 | 인력의 프로젝트 배치 비율 (%) |
| 병목 | 가동률 90% 이상으로 리소스 부족 상태 |
| 파이프라인 | 잠재 프로젝트/기회 목록 |
| ROI | Return on Investment, 투자 대비 수익률 |
| 환각 | AI가 근거 없이 생성한 정보 (Hallucination) |
| 에스컬레이션 | 상위 결재자로 승인 요청 이관 |
